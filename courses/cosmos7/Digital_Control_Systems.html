Understanding the DC Motor

Test 1: How does a DC motor respond to a voltage?

In this test you will learn how a DC motor works. We will first let the motor run freely. Make sure the cables on the pulley are taped to the pulley so that the pulley can freely rotate.

  1. On the ARDUINO program, select Control to be on the on the Open Loop position. Click the Start Loop button.

  2. Slowly move the slider labelled Target and observe the results on the motor. Click the STOP button when you’re finished.

  3. Position the potentiometer on your device pointing up and clear the check box on Target. Click the Start Loop button.

  4. Slowly move the potentiometer on your device and observe the results on the motor. Click the STOP button when you’re finished.

Discussion

  1. Which quantity in the measurement box is directly controlled by the target slider and potentiometer?
  2. The Control measurement indicates what percentage of a maximum available voltage is applied on the motor. Based on your experience with the device so far, how does the motor respond to changes in the applied voltage?
  3. How does the motor respond to small control voltages? Can you explain it?
  4. How does the motor respond to large control voltages? Can you explain it?

Test 2: The static characteristic of the DC motor

You will now perform some experiments to characterize the static behavior of the DC motor. Instead of performing the experiment by hand, you will use a program that will vary the voltage on the motor and then collect and analyze the results.

  1. Click the button Run... and load the program IncrementalStepsOpenLoop.cmd. Wait until the program finishes running.

  2. Click the button Plot... and visualize the measurements collected by the program.

  3. Click the button Spreadsheet.... Clear the check box on only measurements to reveal the data you just collected. Import the data into Excel.

Discussion

  1. Use Excel to generate a scatter plot of Control versus Velocity.
  2. Discuss the results with the other groups. Is the response of the motor linear? How does the motor responds to small voltages? How does the motor responds to large voltages?

A first encounter with feedback

Test 3: Can you regulate velocity?

  1. On the ARDUINO program, select Control to be on the on the Open Loop position.

  2. Position the potentiometer on your device pointing up and clear the check box on Target. Make sure the box Filter velocity is checked. Click the Start Loop button.

  3. Position the potentiometer so that the motor reaches a velocity of <math>4000</math> counts/s. Click the Reset button.

  4. Have a friend touch the top of the big pulley with his or her finger and apply a gentle force trying to stop the motor. Make sure the force does not slow down too much or completely stop the motor, as this may damage the motor. Have your friend remove the finger. Note what happens with the velocity.

  5. Click the STOP button and export the data to Excel using the Spreadsheet. Click the button Plot... to better visualize the experiment.

  6. Repeat steps (a-e) but this time adjust the potentiometer as your friend apply the force so as to try to keep the velocity constant at <math>4000</math> counts/s.

Discussion

  1. What happened to the velocity as your friend touched the pulley?
  2. Use the data you exported to Excel to compute the average and standard deviation of the velocity. Compute the average and standard deviation of the control voltage.
  3. Is the average close to <math>4000</math> counts/s? Compare the variance of the velocity and the control control voltage for a run in which you were adjusting the potentiometer and a run in which you were not adjusting the potentiometer.
  4. Explain in words what you did in order to regulate the voltage at <math>4000</math> counts/s.
  5. Can you formulate your strategy as an algorithm?

Motor Dynamics and Gains

Test 5: The dynamic characteristic of the DC motor

  1. On the ARDUINO program, select Control to be on the on the Open Loop position.

  2. Click the button Run... and load the program SingleStep7680.cmd. Wait until the program finishes running.

  3. Click the button Plot... to visualize the measurements collected by the program.

  4. Click the button Spreadsheet.... Clear the check box on only measurements to visualize the data you just collected. Import the data into Excel.

Discussion

  1. Use Excel to generate a scatter plot of Velocity versus Time.
  2. Compute the average steady-state value of the velocity. Use the average steady-state value of the velocity to compute the approximate motor open loop gain (<math>G</math>).
  3. Compute how many seconds are needed for the motor to reach <math>63</math> % of the steady state value. This is the motor’s open-loop time-constant (<math>\tau</math>).
  4. Can you think of other phenomena that have a similar dynamic characteristic? What about different time-constants? Name a dynamic phenomenon that has a smaller time-constant. Name a dynamic phenomenon that has a much larger time-constant.

Test 6: The dynamic characteristic of the DC motor in closed loop

  1. On the ARDUINO program, select Control to be on the on the Velocity position. Press the Reset button, select a proportional gain of <math>K = 0.25</math> and zero the integral and the derivative gains.

  2. Click the button Run... and load the program SingleStep10240.cmd. Wait until the program finishes running.

  3. Click the button Plot... to visualize the measurements collected by the program.

  4. Click the button Spreadsheet.... Clear the check box on only measurements to visualize the data you just collected. Import the data into Excel.

  5. Repeat steps (a-d) for <math>K = 0.5</math> and <math>K = 1</math>.

Discussion

  1. Use Excel to generate a scatter plot of Velocity versus Time.
  2. Compute the average steady-state value of the velocity. Divide the target velocity by the average steady-state value of the velocity to compute the approximate motor closed-loop gain (<math>H</math>).
  3. Compute how many seconds are needed for the motor to reach <math>63</math> % of the steady state value. This is the motor’s closed-loop time-constant (<math>\rho</math>).
  4. Compare those constants with the open-loop motor constants. Is it faster? Is it slower? Can you explain how the controller did it by looking at the control plot?
  5. After you run the program, locate on your ARDUINO program the values of the controller proportional gain (<math>K</math>) and the motor gain (<math>M</math>).
  6. Compute the quantities <math>\begin{aligned}
   &\frac{K G}{M + K G} &
   &\frac{M \tau}{M + K G}
 \end{aligned}</math> where <math>G</math> is the open loop gain and <math>\tau</math> is the open loop time-constant. Compare these quantities with the closed loop gain <math>H</math> and the closed loop time constant <math>\rho</math>.
  1. How did the choice of gain affected the closed loop gain and time-constant? What value of <math>K</math> would make the closed loop gain <math>\rightarrow 1</math>? What the corresponding value of the closed loop time-constant <math>\rho</math> be? Do you think it is possible to implement such a gain?
  2. Do you think that the other phenomena you thought have a similar dynamic characteristic can be modified through feedback as you did with the motor? What would you need to be able to implement a feedback loop?

Test 7*: Can you beat the motor time-constant?

  1. On the ARDUINO program, select Control to be on the on the Open Loop position.

  2. Position the potentiometer on your device pointing up and clear the check box on Target. Click the Start Loop button.

  3. Experiment with the potentiometer and “learn” where the potentiometer has to be positioned for the motor to experience 50 % of the control voltage.

  4. Move the potentiometer back to the up position (0 % control voltage) and press the Reset button.

  5. Move the potentiometer so as to make the control voltage reach and stay at <math>50</math> %.

  6. Look at the plot and the spreadsheet to calculate your time-constant. Can you make it faster than the time-constant you computed in the previous tests?

Test 8*: Setting a zero target

  4. Repeat steps (a-c) for <math>K = 0.5</math> and <math>K = 1</math>.

Test 12: Closing the loop

  1. On the ARDUINO program, select Control to be on the on the Position position. Press the Reset button, select a proportional gain of <math>K = 0.1</math> and zero the integral and the derivative gains.

  2. Set the check box on Target and move the slider to <math>3000</math> counts. Click the Reset button then the Start Loop button.

  3. Wait until the pulley is no longer moving then click the STOP button and export the data to Excel using the Spreadsheet.

  4. Repeat steps (a-c) for <math>K = 0.2</math>, <math>K = 0.5</math>.

Discussion

  1. Use Excel to generate a scatter plot of Velocity versus Time.
  2. How different is this type of response to the one you observed when controlling velocity?
  3. Compute the average steady-state value of the position. Divide the target position by the average steady-state value of the position to compute the approximate motor closed-loop gain (<math>H</math>).
  4. Compute how many seconds are needed for the motor to go from <math>10
 \%</math> of the target position to <math>90 \%</math>. This is the motor’s closed-loop rise-time (<math>\sigma</math>).
  1. Can you explain how we obtained a unitary gain without integral control?

Test 15: Moving a load

Make sure this is the last experiment you do. For this experiment you will need to release the fishing line and its hook from the large pulley.

  1. On the ARDUINO program, select Control to be on the on the Position position. Press the Reset button, select a proportional gain of <math>K = 0.2</math> and zero the integral and the derivative gains.

  2. Attach a bag of marbles with approximately <math>0.5</math> Kg (the load) to the string hook. Make sure the hook is position as close as possible to the pulley and click the Reset button. If needed, manually wind up the fishing line.

  3. Set the check box on Target and move the slider to <math>3000</math> counts then click the Start Loop button.

  4. Wait until the pulley is no longer moving then click the STOP button and export the data to Excel using the Spreadsheet.

  5. Repeat steps (a-d) for <math>K = 0.5</math>, <math>K = 1</math>.

Discussion

  1. How does the load affect the behavior of the system?
  2. For the system to perform as well as in the unloaded case, how do you have to modify the control gain?

Test 16*: Integral control?

  1. Repeat the experiment in Test 13 but this time make sure that you put enough marbles in the bag so that the motor starts moving when the controller is off. Be careful before you turn off the controller as the bag of marbles will simply drop.

Discussion

  1. How do you explain the observed behavior?
  2. Can integral control fix the problem once again?
